Fix async execution validation to prevent TypeError in record mode

This commit fixes the critical bug where record mode message processing
failed with "TypeError: An asyncio.Future, a coroutine or an awaitable
is required". The issue was caused by lack of input validation in the
_run_async_with_timeout() method.

## Problem
- All record mode messages failed to process
- TypeError occurred when invalid inputs reached asyncio internals
- Messages retried 3 times before final failure
- No notes saved to database or displayed on web

## Root Cause
- _run_async_with_timeout() did not validate if input was a coroutine
- Event loop state was not checked before execution
- Error messages from asyncio were unclear and hard to debug

## Solution
1. Added coroutine/awaitable type validation using asyncio.iscoroutine()
   and hasattr(coro, '__await__')
2. Added event loop state check (exists and not closed)
3. Enhanced TypeError handling in _execute_with_flood_retry()
4. Provide clear error messages with actual type information
